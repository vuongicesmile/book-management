Cache — tránh query DB và gọi OpenAI lặp lại

Vấn đề không có cache

GET /books/42 → DB query (5ms) → response GET /books/42 → DB query (5ms) → response ← query giống hệt, tốn DB thêm GET /books/42 → DB query (5ms) → response ← 100 users cùng lúc = 100 queries POST /ai/books/42/summarize → OpenAI (3s, $0.001) POST /ai/books/42/summarize → OpenAI (3s, $0.001) ← cùng book, cùng nội dung = lãng phí

Flow với cache

GET /books/42 ↓ cache_get("bm:book:42") HIT → return immediately (0ms) MISS → DB query → cache_set("bm:book:42", data, ttl=300) → return POST /ai/books/42/summarize ↓ cache_get("bm:ai:summary:42") HIT → return cached summary (0ms, $0) MISS → OpenAI call → cache_set(..., ttl=3600) → return

app/common/redis.py — building blocks

# cache_key: namespaced keys — học từ vuonglearning "ilmu:user:42"
cache_key("book", 42)       # → "bm:book:42"
cache_key("ai", "summary", 42) # → "bm:ai:summary:42"

# cache_get: JSON deserialize, None nếu miss hoặc Redis down (fail-open)
cached = await cache_get(key)

# cache_set: JSON serialize + TTL
await cache_set(key, data, ttl=300)

# cache_delete: invalidate khi data thay đổi
await cache_delete(key)  # gọi sau PUT/DELETE

Áp dụng — GET /books/{id}

# app/api/endpoints/books.py
@router.get("/{book_id}")
async def get_book(book_id: int, db = Depends(get_db)):
    key = cache_key("book", book_id)
    cached = await cache_get(key)
    if cached:
        return cached               # HIT: 0ms, no DB

    book = BookRepository(db).get_by_id(book_id)   # MISS: query DB
    await cache_set(key, {...}, ttl=settings.cache_book_ttl)
    return book

@router.put("/{book_id}")
async def update_book(book_id: int, ...):
    book = BookRepository(db).update(book_id, ...)
    await cache_delete(cache_key("book", book_id))  # INVALIDATE
    return book

Áp dụng — AI result cache

# app/ai/service.py — summarize
async def summarize(book_id, db):
    key = cache_key("ai", "summary", book_id)
    cached = await cache_get(key)
    if cached:
        return SummarizeResponse(**cached)   # HIT: $0, 0ms

    # MISS: gọi OpenAI
    summary = await tasks._call_chat(prompt, ...)
    result = SummarizeResponse(...)
    await cache_set(key, result.model_dump(), ttl=3600)  # 1h
    return result
💡
TTL strategy: Book data (5 phút) — ít thay đổi nhưng có thể update.
AI results (1 giờ) — cùng book cùng nội dung, tốn tiền gọi lại.
Khi user PUT /books/{id} → invalidate cả book cache + AI cache.

Rate Limiting — bảo vệ OpenAI cost

Tại sao cần rate limit AI endpoints?

  • Mỗi call OpenAI tốn tiền — không limit → abuse → bill tăng đột biến
  • vuonglearning limit: signin=10/window, default=220/window
  • Book-management: AI endpoints = 20 calls/60s per IP

Lua script — atomic, không race condition

# app/common/rate_limit.py
# Học trực tiếp từ vuonglearning rate_limit.py:88
_RATE_LIMIT_LUA = """
local current = redis.call('INCR', KEYS[1])
if current == 1 then
    redis.call('EXPIRE', KEYS[1], ARGV[1])
end
return current
"""

# Tại sao Lua thay vì INCR + EXPIRE riêng?
# Race condition nếu tách:
#   Process A: INCR key → 1
#   Process B: INCR key → 2
#   Process A: EXPIRE key 60  ← set TTL
#   Process B: EXPIRE key 60  ← reset TTL! window bị kéo dài
# Lua: 2 lệnh chạy atomically = không bị interrupt giữa chừng

Áp dụng vào router

# app/ai/router.py
@router.post("/books/{book_id}/summarize")
async def summarize(book_id: int, request: Request, db = Depends(get_db)):
    await check_ai_rate_limit(request)   # raise 429 nếu > 20/60s
    return await service.summarize(book_id, db)

Flow khi vượt limit

IP 192.168.1.1 → POST /ai/books/1/summarize → counter=1 ✓ IP 192.168.1.1 → POST /ai/books/2/summarize → counter=2 ✓ ... IP 192.168.1.1 → POST /ai/books/20/summarize → counter=20 ✓ IP 192.168.1.1 → POST /ai/books/21/summarize → counter=21 ✗ 429 Sau 60s: key tự expire → counter reset → lại cho phép
★
Fail-open: Nếu Redis down → cho request qua, không block user. Học từ vuonglearning: availability quan trọng hơn strict rate enforcement.

Async Queue (bài 154–162) — RAG indexing không timeout

Vấn đề: PDF indexing mất 30–120 giây

SYNC (bad): POST /ai/rag/books/1/index → load PDF → split 1200 chunks → embed × 1200 → ChromaDB |←─────────────────────────────── 90 giây ────────────────────────────────────────→| Nginx timeout sau 30s → 504 Gateway Timeout → kết quả mất ASYNC (good): POST /ai/rag/books/1/index/async → enqueue job → {"job_id": "abc-123"} |←────── <100ms ──────────────────→| [Worker process riêng — không có HTTP connection, không có timeout] Worker: load PDF → split → embed × 1200 → ChromaDB → save result |←─────────────────────────── 90 giây thoải mái ──────────────────────────────→| GET /ai/rag/jobs/abc-123 → {"status": "started"} GET /ai/rag/jobs/abc-123 → {"status": "finished", "result": {...}}

3 components

# app/queue.py — RQ Queue singleton
redis_conn = Redis.from_url(settings.redis_url)   # sync Redis (RQ không dùng asyncio)
rag_queue = Queue(settings.rag_queue_name, connection=redis_conn)

# app/ai/rag/tasks.py — function chạy trong worker
def index_book_pdf_task(book_id, file_bytes, filename) -> dict:
    # asyncio.run() vì service là async def, worker là sync process
    result = asyncio.run(service.index_book_pdf(book_id, file_bytes, filename, db))
    return result.model_dump()

# worker.py — entry point, chạy song song với FastAPI
worker = Worker([Queue("rag-indexing", conn)], connection=conn)
worker.work(with_scheduler=True)

Enqueue endpoint

# app/ai/rag/router.py
@router.post("/books/{book_id}/index/async", status_code=202)
async def index_book_async(book_id, file: UploadFile):
    file_bytes = await file.read()
    job = get_rag_queue().enqueue(
        index_book_pdf_task,          # importable function
        book_id, file_bytes, file.filename,
        job_timeout=300,              # worker có 5 phút
        result_ttl=3600,              # giữ result 1h
    )
    return {"job_id": job.id, "status": "queued"}

Poll endpoint

@router.get("/jobs/{job_id}")
def get_job_status(job_id):
    job = Job.fetch(job_id, connection=get_redis_sync())
    status = job.get_status()

    if status == JobStatus.FINISHED:
        return {"status": "finished", "result": job.result}
    if status == JobStatus.FAILED:
        return {"status": "failed", "error": job.latest_result().exc_string}
    return {"status": status.value}   # "queued" | "started"
