Memory Layer in AI Agents

Bài 177–188 Theory Mem0 Vector DB Cách AI nhớ thông tin người dùng qua nhiều cuộc hội thoại.

1. Tại sao Agent cần Memory?

Không có memory

Hôm nay: User: "Tôi thích sách Python" AI: "Hay nhé, tôi sẽ gợi ý sách..." Ngày mai: User: "Gợi ý sách cho tôi" AI: "Bạn thích thể loại gì?" ← Không nhớ gì cả

Có memory

Hôm nay: User: "Tôi thích sách Python" AI: "OK, tôi nhớ rồi." [memory: "Thích sách Python"] Ngày mai: User: "Gợi ý sách cho tôi" AI: "Dựa trên sở thích của bạn về Python, tôi gợi ý..." ← Nhớ đúng, personalized

Memory = khả năng agent lưu và truy xuất thông tin vượt qua giới hạn của một cuộc hội thoại (context window).

Không có memory → mỗi chat là "người lạ" mới. Có memory → AI hiểu người dùng theo thời gian.

2. Các loại Memory Architecture (Bài 178–184)

Short-Term Memory In-context

Conversation history trong 1 chat session. Nằm trong context window.

# Là messages list gửi cho LLM
messages = [
  {"role": "system", ...},
  {"role": "user", "content": "Tôi thích Python"},
  {"role": "assistant", ...},
  {"role": "user", "content": "Gợi ý sách"},
]
# LLM đọc toàn bộ → nhớ trong session

⚡ Nhanh, free — nhưng chỉ tồn tại trong session hiện tại. Chat mới = mất hết.

Long-Term Memory External Storage

Lưu ngoài vào DB/vector store, tồn tại mãi mãi (hoặc cho đến khi user xóa).

# Trước mỗi chat → fetch từ DB
memory = db.get_user_memory(user_id)
system_prompt = f"User info: {memory}"

# Sau mỗi chat → update DB
db.save_memory(user_id, new_facts)

💾 Persistent cross-session. Cần DB. Cần chọn cái gì đáng nhớ.

Factual Memory Facts & Profiles

Sự kiện cụ thể về người dùng. Ví dụ: "tên là Vuong", "thích Python", "làm ở YTL".

facts = [
  "Name is Vuong",
  "Works at YTL AI Labs",
  "Prefers Python over Java",
]

📌 Source of truth. Discrete. Dễ edit từng fact.

Episodic Memory Past Events

Lịch sử sự kiện đã xảy ra. Ví dụ: "hôm qua hỏi về Django", "tuần trước tìm sách ML".

episodes = [
  {"date": "2026-06-20", "topic": "Django"},
  {"date": "2026-06-18", "topic": "ML books"},
]

📅 Giúp agent hiểu context theo thời gian.

Semantic Memory Vector Search

Lưu dạng vector embedding. Tìm kiếm theo nghĩa, không phải từ khóa.

# Query: "Python programming"
# Tìm gần nhất trong vector space:
# → "Sách về lập trình Python" ✓
# → "Học coding với Python" ✓
# → "Java tutorial" ✗ (xa)

🔍 RAG pattern. Scalable. Phù hợp knowledge base lớn.

3. Short-Term Memory — Quản lý Context Window (Bài 180)

Vấn đề: Context window có giới hạn

GPT-4o-mini: 128,000 tokens (~100,000 từ) Claude Sonnet: 200,000 tokens Conversation dài → vượt giới hạn → API lỗi hoặc tốn tiền. Giải pháp: cắt bớt messages cũ (truncation) hoặc tóm tắt (summarization).

Truncation

# Giữ N messages gần nhất
MAX_HISTORY = 20

def trim_messages(messages):
    system = [m for m in messages
              if m["role"] == "system"]
    history = [m for m in messages
               if m["role"] != "system"]
    return system + history[-MAX_HISTORY:]

✅ Đơn giản. ❌ Mất context cũ hoàn toàn.

Summarization (Rolling)

# Tóm tắt messages cũ thành 1 đoạn
if token_count > THRESHOLD:
    old = messages[:-10]  # messages cũ
    summary = llm.summarize(old)
    messages = [
      system_msg,
      {"role": "system",
       "content": f"Earlier: {summary}"},
      *messages[-10:]   # giữ 10 gần nhất
    ]

✅ Giữ tinh túy. ❌ Tốn thêm LLM call.

Mem0 là thư viện Python giúp agent có long-term memory. Tự động extract facts từ conversation và lưu vào vector DB.

Tích hợp vào Agent

async def chat_with_memory(user_id: str, user_message: str) -> str:
    # 1. Fetch relevant memories trước khi chat
    memories = m.search(query=user_message, user_id=user_id)
    memory_context = "\n".join(r["memory"] for r in memories)

    # 2. Inject vào system prompt
    system = f"""Bạn là AI assistant.
Những gì bạn biết về người dùng:
{memory_context}"""

    # 3. Gọi LLM
    response = await llm.chat([
        {"role": "system", "content": system},
        {"role": "user", "content": user_message},
    ])

    # 4. Update memory sau khi chat
    m.add(messages=[
        {"role": "user", "content": user_message},
        {"role": "assistant", "content": response},
    ], user_id=user_id)

    return response

5. Vector Database — Semantic Memory (Bài 187–188)

Tại sao Vector DB?

SQL search (keyword):

WHERE fact LIKE '%Python%'
# Tìm "Thích lập trình Python" ✓
# Không tìm được "Coding enthusiast" ✗

Vector search (meaning):

query_vector = embed("Python programmer")
nearest = db.search(query_vector, top_k=3)
# "Thích Python" ✓ (score: 0.95)
# "Coding enthusiast" ✓ (score: 0.82)

Qdrant Python client

from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams, PointStruct
import openai

client = QdrantClient(host="localhost", port=6333)

# Tạo collection
client.create_collection(
    collection_name="user_memories",
    vectors_config=VectorParams(
        size=1536,          # OpenAI text-embedding-3-small
        distance=Distance.COSINE,
    ),
)

# Lưu memory với vector
def embed(text: str) -> list[float]:
    resp = openai.embeddings.create(
        input=text, model="text-embedding-3-small"
    )
    return resp.data[0].embedding

client.upsert(
    collection_name="user_memories",
    points=[PointStruct(
        id=1,
        vector=embed("Thích lập trình Python"),
        payload={"text": "Thích lập trình Python", "user_id": "u1"}
    )],
)

# Semantic search
hits = client.search(
    collection_name="user_memories",
    query_vector=embed("ngôn ngữ lập trình"),
    query_filter={"must": [{"key": "user_id", "match": {"value": "u1"}}]},
    limit=5,
)
# → [ScoredPoint(id=1, score=0.94, payload={"text":"Thích lập trình Python"})]

6. Vuonglearning — Memory Architecture (Thực tế Production)

Lịch sử: Mem0 → Custom Hybrid System

Giai đoạn 1 (cũ): Dùng Mem0 - Vector DB (pgvector) + Mem0 library - Tự động extract facts bằng Mem0 - Semantic search để retrieve Giai đoạn 2 (hiện tại): Custom Hybrid v3 - Bỏ Mem0 → tự build (migration f7a1b2c3d4e5) - Lý do: cần control privacy (health data), custom categories, PII filter - Architecture 2 tầng: Facts (discrete) + Summary (synthesized)

Kiến trúc hiện tại: 2-Layer Hybrid

Layer 1: Facts (Discrete)

Bảng user_memory_facts — mỗi fact là 1 row

{
  "text": "Works at YTL AI Labs",
  "category": "identity",
  "source": "explicit",  # user said "remember"
  "confidence": 0.95
},
{
  "text": "Prefers Python",
  "category": "preference",
  "source": "extracted",  # auto background
  "confidence": 0.80
}

5 categories: identity, preference, project, context, relationship

Layer 2: Summary (Synthesized)

Bảng user_memories — 1 row per user

"summary": """User works at YTL AI Labs
as a software engineer. Prefers
Python and is currently learning
AI agent architecture. Has been
asking about Redis and book
management systems."""

# Generated FROM facts by LLM
# Injected into system prompt

Max 2000 chars. Injected vào mỗi chat. Regenerated sau khi facts thay đổi.

Flow hoàn chỉnh trong 1 cuộc hội thoại

── Trước khi chat ────────────────────────────────────────────── 1. retrieve_memory_summary(user_id) → fetch user_memories.summary từ DB (timeout 2s, fail-open) 2. Inject vào system prompt: "Bạn biết những điều này về người dùng: {summary}" ── Trong khi chat ────────────────────────────────────────────── 3. User: "Nhớ rằng tôi thích sách Python" → LLM gọi tool: remember_this(fact="Thích sách Python", category="preference") → POST /internal/v1/users/{id}/memories/facts → Lưu vào user_memory_facts 4. Background: auto-extract facts từ conversation (chạy ngầm, không block response) ── Sau khi chat ──────────────────────────────────────────────── 5. _rewrite_memory_background() — background task: a. Đọc tất cả facts của user b. Gọi LLM: "Viết tóm tắt từ các facts này" c. Update user_memories.summary (optimistic concurrency) ── Chat tiếp theo ────────────────────────────────────────────── 6. Summary mới được inject vào chat mới → AI nhớ ngữ cảnh

9. Quick Reference — Chọn loại Memory nào?

if cần nhớ trong session hiện tại: → Short-term (messages list) # free, đã có sẵn elif cần nhớ cross-session, đơn giản: → Summary in PostgreSQL # text column, inject vào system prompt elif cần facts riêng lẻ, user có thể edit: → Discrete facts table # như vuonglearning user_memory_facts elif cần tìm kiếm theo nghĩa (semantic search): → Vector DB (Qdrant / pgvector) # dùng cho knowledge base lớn elif muốn all-in-one, không tự build: → Mem0 library # auto extract + vector search elif production với privacy requirements: → Custom hybrid # như vuonglearning: facts + summary + filters
