Bài 207-208

Model Context Protocol (MCP)

Chuẩn giao tiếp mở giữa AI models và external tools / data sources

1. MCP là gì? (Bài 207)

Model Context Protocol (MCP) là một open standard do Anthropic phát hành năm 2024. Nó định nghĩa cách một AI model "nói chuyện" với external systems (databases, GitHub, Slack, filesystem, v.v.) thông qua một giao thức chuẩn hóa.

Tư tưởng cốt lõi: Trước MCP, mỗi AI app phải tự tích hợp riêng với từng tool — custom code, custom format, custom auth. MCP giống như USB-C cho AI: một cổng chuẩn để kết nối bất kỳ AI nào với bất kỳ data source nào.

Vấn đề MCP giải quyết

❌ Trước MCP

Mỗi app tự build integration:

ChatApp_A → GitHub API (custom)
ChatApp_B → GitHub API (custom)
ChatApp_C → GitHub API (custom)
# 3 apps × N tools = 3N integrations

✅ Với MCP

Một server, nhiều clients dùng:

ChatApp_A ─┐
ChatApp_B ─┼─→ MCP Server (GitHub)
ChatApp_C ─┘
# N apps × 1 server = 1 integration

MCP không phải là...

Khái niệm dễ nhầmSự thật
Function calling / tool use Function calling là cách LLM quyết định gọi tool. MCP là cách tool được expose cho LLM. MCP thường dùng qua function calling.
Plugin system MCP là transport protocol. Plugin system có thể build trên MCP, nhưng chúng không giống nhau.
API MCP là protocol hai chiều (bidirectional), không phải REST API một chiều.
LangChain / LangGraph LangChain là orchestration framework. MCP là transport layer. Có thể dùng cả hai cùng nhau.

2. Kiến trúc MCP (Bài 208)

MCP có 3 thành phần chính: Host, Client, và Server.

┌─────────────────────────────────────────────────────────────────┐ │ HOST │ │ (Claude Desktop, Claude Code, VS Code, custom AI app...) │ │ │ │ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │ │ │ MCP Client │ │ MCP Client │ │ MCP Client │ │ │ │ (1) │ │ (2) │ │ (3) │ │ │ └──────┬───────┘ └──────┬───────┘ └──────┬───────┘ │ │ │ │ │ │ └─────────┼───────────────────┼───────────────────┼─────────────┘ │ JSON-RPC 2.0 │ JSON-RPC 2.0 │ JSON-RPC 2.0 │ (stdio / SSE) │ (stdio / SSE) │ (stdio / SSE) ▼ ▼ ▼ ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐ │ MCP Server │ │ MCP Server │ │ MCP Server │ │ (GitHub) │ │ (PostgreSQL) │ │ (Filesystem) │ │ │ │ │ │ │ │ Tools: │ │ Tools: │ │ Resources: │ │ - search_repos │ │ - run_query │ │ - read_file │ │ - create_issue │ │ - list_tables │ │ - list_dir │ │ - get_pr │ │ - describe_db │ │ - write_file │ └─────────────────┘ └─────────────────┘ └─────────────────┘

Host

Ứng dụng chứa AI model. Ví dụ: Claude Desktop, Claude Code, VS Code Copilot. Host quản lý vòng đời của các MCP clients và quyết định tools nào được expose cho model.

MCP Client

Thành phần chạy bên trong Host. Mỗi client duy trì một kết nối 1:1 với một MCP Server. Client xử lý protocol handshake, capability negotiation, và message routing.

MCP Server

Process nhỏ (có thể là Node.js, Python, Rust...) expose ba loại tài nguyên:

LoạiMô tảVí dụ
Tools Hàm có thể gọi (model-controlled). LLM quyết định khi nào gọi. search_repos, run_query, send_message
Resources Dữ liệu có thể đọc (application-controlled). Host quyết định khi nào đọc. File contents, database rows, API responses
Prompts Templates đã định sẵn (user-controlled). Người dùng chọn dùng. "Review this PR", "Explain this SQL"

Transport: JSON-RPC 2.0

Mọi giao tiếp trong MCP dùng JSON-RPC 2.0 — format đơn giản, stateless, dễ debug. Hai loại transport:

stdio (local)

Host spawn MCP Server dưới dạng child process.
Giao tiếp qua stdin/stdout.
✓ Zero network overhead
✓ Simple, secure (local only)
○ Chỉ local

SSE / HTTP (remote)

Host kết nối đến remote MCP Server qua HTTP + SSE.
Server gửi events, client gửi POST requests.
✓ Remote servers
✓ Multi-client, shared state
○ Cần auth, network

Lifecycle của một tool call qua MCP

User: "Tìm PR nào đang open trong repo my-project" │ ▼ Host (Claude Code) → LLM nhận prompt + tool list từ MCP Server (GitHub) → LLM quyết định gọi: search_pull_requests({repo: "my-project", state: "open"}) │ ▼ JSON-RPC request qua stdio MCP Server (GitHub) → Validate params → Gọi GitHub REST API → Format kết quả → Trả về JSON-RPC response │ ▼ Host (Claude Code) → Inject tool result vào conversation → LLM đọc kết quả → viết câu trả lời cho user

3. Vuonglearning dùng MCP như thế nào?

Phân biệt quan trọng: Có hai ngữ cảnh hoàn toàn khác nhau khi nói về MCP trong dự án này.

A. Môi trường phát triển (Claude Code) — CÓ dùng MCP

Claude Code (tool AI dùng để viết code cho vuonglearning) cấu hình các MCP servers trong ~/.claude/settings.json và .claude/settings.local.json. Đây là công cụ cho developer, không phải cho end user.

MCP ServerTransportTools exposeDùng để
plugin_ytlailabs_github SSE get_pull_request, create_issue, search_code... Claude Code đọc PR, tạo issue, tìm code trong repo
plugin_ytlailabs_playwright SSE browser_navigate, browser_screenshot... E2E testing, UI verification trong quá trình dev
plugin_ytlailabs_context7 SSE resolve-library-id, query-docs Tra cứu docs của FastAPI, SQLAlchemy, Next.js khi code
plugin_ytlailabs_exa SSE web_search_exa, web_fetch_exa Web search khi cần tìm solutions, examples

B. Runtime của app (vuonglearning product) — KHÔNG dùng MCP

Vuonglearning (ứng dụng chat AI cho người dùng cuối) không dùng MCP protocol ở runtime. Thay vào đó, nó có hệ thống tool riêng:

vuonglearning runtime architecture: ai-service └── agents/ ├── chat/agent.py ← ChatAgent (5300+ lines) ├── registry.py ← AGENT_REGISTRY (custom) └── tools/ ├── web_search_tool.py ← tích hợp SerpAPI trực tiếp ├── calculator_tool.py ├── education_quiz_tool.py └── ... Tool call flow: User message → ChatAgent._tool_use_loop() ← vòng lặp giống bài 163 → OpenAI function calling ← LLM quyết định gọi tool → tool_fn(args) ← Python function, KHÔNG qua MCP → kết quả inject vào messages → lặp lại cho đến finish_reason="stop"
Tại sao vuonglearning không dùng MCP ở runtime?

MCP tốt khi cần tích hợp nhiều loại external systems, reuse across multiple AI clients, hoặc expose tools cho third-party. Vuonglearning có tool set cố định, tất cả internal — custom tool registry đơn giản và nhanh hơn. Không cần overhead của MCP protocol khi tools không được share ra ngoài.

4. MCP vs Custom Tool Registry

MCP Custom Registry (book-management)
Portability Cao — bất kỳ MCP-compatible host nào cũng dùng được Thấp — tied to app cụ thể
Setup Phức tạp hơn — server process, JSON-RPC, transport Đơn giản — Python decorator + dict
Performance Thêm serialization overhead (JSON-RPC) Direct function call — nhanh nhất
Ecosystem Cộng đồng lớn — 1000+ MCP servers có sẵn Chỉ tools bạn tự viết
Khi nào dùng Cần reuse tools across nhiều AI clients, hoặc external tools Internal-only tools, một AI app duy nhất
Ví dụ Claude Code + GitHub, Claude Code + PostgreSQL book-management BookAgent tools

5. Build một MCP Server đơn giản (Python)

Để hiểu MCP sâu hơn, đây là cách tạo một MCP Server nhỏ bằng Python SDK chính thức:

# pip install mcp
from mcp.server.fastmcp import FastMCP
from sqlalchemy.orm import Session

# Tạo MCP server với tên "book-library"
mcp = FastMCP("book-library")

# Expose tool — giống @register_tool nhưng theo chuẩn MCP
@mcp.tool()
def search_books(query: str) -> list[dict]:
    """Tìm sách theo từ khóa trong tiêu đề hoặc mô tả."""
    db: Session = get_db_session()
    books = db.query(Book).filter(
        Book.title.ilike(f"%{query}%")
    ).limit(10).all()
    return [
        {"id": b.id, "title": b.title, "author": b.author}
        for b in books
    ]

# Expose resource — dữ liệu đọc được (không phải function call)
@mcp.resource("book://{book_id}")
def get_book_resource(book_id: int) -> str:
    """Lấy nội dung đầy đủ của sách theo ID."""
    db: Session = get_db_session()
    book = db.get(Book, book_id)
    return f"Title: {book.title}\nAuthor: {book.author}\nDesc: {book.description}"

# Chạy server qua stdio (để dùng với Claude Code)
if __name__ == "__main__":
    mcp.run(transport="stdio")

Để Claude Code dùng server này, thêm vào ~/.claude/settings.json:

{
  "mcpServers": {
    "book-library": {
      "command": "python",
      "args": ["/path/to/mcp_server.py"],
      "env": {
        "DATABASE_URL": "sqlite:///books.db"
      }
    }
  }
}

6. Tổng kết — Khi nào dùng MCP?

Scenario Recommend
Tool dùng cho nhiều AI clients khác nhau (Claude Code, VS Code, custom app) MCP Server
Tích hợp external service đã có MCP server (GitHub, Postgres, Slack) Dùng existing MCP Server
Tool chỉ dùng trong một AI app cụ thể Custom Tool Registry
Học/prototype — muốn đơn giản Custom Tool Registry
Production app cần expose tools ra cho developers MCP Server
Key takeaway: MCP giải quyết vấn đề N × M integrations (N AI clients × M tools). Nếu bạn chỉ có 1 app với tools internal, custom registry đơn giản và đủ. Khi cần scale hoặc share, MCP là con đường đúng.
