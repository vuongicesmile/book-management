Understanding Large Language Models (LLMs)

LLM là model được train trên lượng text khổng lồ để học xác suất của từ tiếp theo. Nó không "hiểu" ngôn ngữ — nó dự đoán token tiếp theo dựa trên context.

Analogy: LLM giống một người đã đọc toàn bộ internet. Khi bạn hỏi "Thủ đô của Pháp là...", nó biết từ tiếp theo là "Paris" vì đã thấy pattern đó hàng triệu lần trong training data.

Cách LLM hoạt động (đơn giản nhất)

Input: "Hôm nay trời" ↓ [Tokenize] → [103, 45, 872] ↓ [LLM predict next token] ↓ Output: "đẹp" (probability: 0.42) "mưa" (probability: 0.31) "nóng" (probability: 0.18) ... ↓ Chọn "đẹp" → lặp lại với "Hôm nay trời đẹp"
Ví dụ LLMs
GPT-4, Claude, Gemini, Llama, Mistral
Training data
Hàng nghìn tỷ tokens từ web, books, code

Deep Dive into the GPT Architecture

GPT (Generative Pre-trained Transformer) là Transformer Decoder — chỉ dùng decoder stack, không có encoder. Sinh text từ trái sang phải.

Kiến trúc GPT

Input tokens: ["Hôm", "nay", "trời"] ↓ ┌─────────────────────────────┐ │ Token Embedding │ "Hôm" → vector [0.2, -0.5, 0.8, ...] │ + Positional Encoding │ + vị trí (position 0, 1, 2) └─────────────────────────────┘ ↓ ┌─────────────────────────────┐ │ Transformer Block × N │ GPT-3: N=96 layers │ ┌───────────────────────┐ │ │ │ Multi-Head Attention │ │ ← Bài 110 │ │ Feed Forward Network │ │ │ │ Layer Normalization │ │ │ └───────────────────────┘ │ └─────────────────────────────┘ ↓ ┌─────────────────────────────┐ │ Linear + Softmax │ → probability của 50,000 tokens └─────────────────────────────┘ ↓ Output: "đẹp" (pick highest prob)
GPT-3
175B params, 96 layers, 96 attention heads
GPT-4

Fundamentals of Tokenization in NLP

LLM không đọc chữ — nó đọc tokens (số nguyên). Tokenization = chuyển text thành list số.

Token KHÔNG phải là word

"Hello world" → ["Hello", " world"] → [9906, 1917] "unhappiness" → ["un", "happ", "iness"] → [359, 5316, 1885] "GPT" → ["G", "PT"] → [38, 2898] "🎉" → ["<0xF0>","<0x9F>","<0x8E>","<0x89>"] → 4 tokens!

Byte Pair Encoding (BPE) — thuật toán phổ biến nhất

Bước 1: Bắt đầu từng ký tự riêng lẻ "low", "lower", "newest", "widest" → l, o, w, e, r, n, s, t, i, d Bước 2: Merge cặp xuất hiện nhiều nhất "l"+"o" → "lo" (xuất hiện 2 lần) "lo"+"w" → "low" (xuất hiện 2 lần) Bước 3: Lặp lại cho đến khi đủ vocab size (50,000 tokens)

Tại sao tokenization quan trọng?

  • Token limit: GPT-4 = 128k tokens context window
  • Cost: API tính tiền theo token, không phải chữ
  • Performance: "1+1=2" và "one plus one equals two" có độ dài token khác nhau
  • Language bias: tiếng Anh hiệu quả hơn tiếng Việt (cùng 1 nghĩa nhưng tiếng Việt dùng nhiều token hơn)

Implementing a Custom Tokenizer in Python

Implement simple tokenizer từ đầu để hiểu cơ chế, sau đó dùng tiktoken (tokenizer thực của OpenAI).

Simple word tokenizer

class SimpleTokenizer:
    def __init__(self):
        self.vocab = {}      # word → id
        self.reverse = {}    # id → word
        self.next_id = 0

    def fit(self, texts: list[str]) -> "SimpleTokenizer":
        for text in texts:
            for word in text.lower().split():
                if word not in self.vocab:
                    self.vocab[word] = self.next_id
                    self.reverse[self.next_id] = word
                    self.next_id += 1
        return self

    def encode(self, text: str) -> list[int]:
        return [self.vocab.get(w, -1) for w in text.lower().split()]

    def decode(self, ids: list[int]) -> str:
        return " ".join(self.reverse.get(i, "[UNK]") for i in ids)

# Dùng thử
tokenizer = SimpleTokenizer()
tokenizer.fit(["hôm nay trời đẹp", "hôm nay trời mưa"])

ids = tokenizer.encode("hôm nay trời đẹp")
# → [0, 1, 2, 3]

text = tokenizer.decode(ids)
# → "hôm nay trời đẹp"

The Transformer Breakthrough: "Attention is All You Need"

Paper 2017 của Google. Trước đó AI dùng RNN/LSTM — chậm vì xử lý tuần tự. Transformer xử lý song song toàn bộ sequence cùng lúc.

RNN (cũ): Đọc sách từng chữ một, ghi nhớ qua "hidden state" — chữ đầu tiên dễ bị quên đến cuối câu.

Transformer (mới): Nhìn tất cả chữ cùng lúc, tính "mức độ liên quan" giữa mọi cặp chữ — nhớ được context dài hơn nhiều.

Core idea: Self-Attention

Câu: "Con mèo ngồi trên tấm thảm vì nó mệt" Khi xử lý từ "nó": - "nó" liên quan tới "mèo": 0.92 ← attention cao - "nó" liên quan tới "thảm": 0.03 - "nó" liên quan tới "mệt": 0.71 ← attention cao → Model hiểu "nó" = "mèo" (không phải "thảm")

So sánh RNN vs Transformer

RNN/LSTM
✗ Tuần tự, chậm
✗ Quên context dài
✗ Khó train song song
Transformer
✓ Song song, nhanh
✓ Nhớ toàn bộ context
✓ Scale được lên hàng tỷ params

Deep Diving into Vector Embeddings

Embedding = chuyển text thành vector số trong không gian nhiều chiều. Những thứ có nghĩa tương tự sẽ có vector gần nhau.

Analogy: Trên bản đồ, Hà Nội và Hải Phòng gần nhau (đều miền Bắc). Embedding tương tự — "king" và "queen" gần nhau trong vector space vì nghĩa liên quan.

Ví dụ nổi tiếng: Word Arithmetic

vector("king") - vector("man") + vector("woman") ≈ vector("queen") vector("Paris") - vector("France") + vector("Vietnam") ≈ vector("Hà Nội")

Python: Dùng OpenAI Embeddings

from openai import OpenAI
import numpy as np

client = OpenAI()

def get_embedding(text: str) -> list[float]:
    response = client.embeddings.create(
        input=text,
        model="text-embedding-3-small"
    )
    return response.data[0].embedding
    # → list 1536 số float

def cosine_similarity(a: list[float], b: list[float]) -> float:
    """Đo độ tương đồng: 1.0 = giống nhau, 0.0 = khác nhau."""
    a, b = np.array(a), np.array(b)
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# Ứng dụng thực: tìm sách tương tự
books = [
    "Clean Code - Robert Martin",
    "The Pragmatic Programmer",
    "Design Patterns - Gang of Four",
]
query = "best practices in software development"

query_emb = get_embedding(query)
scores = [(book, cosine_similarity(query_emb, get_embedding(book))) for book in books]
scores.sort(key=lambda x: x[1], reverse=True)
# → [("Clean Code", 0.89), ("Pragmatic...", 0.85), ...]

Ứng dụng trong book-management

# Với embedding: tìm "python web" → tìm được "FastAPI development"
# vì 2 câu này có vector gần nhau trong embedding space

Role of Positional Encodings in Transformers

Transformer xử lý tất cả tokens song song — nên không biết thứ tự mặc định. Positional Encoding thêm thông tin vị trí vào embedding.

Vấn đề: "Vuong loves Python" và "Python loves Vuong" có cùng tokens, khác thứ tự — nghĩa hoàn toàn khác.
Giải pháp: Thêm vector vị trí vào mỗi token — token ở position 0, 1, 2 có "dấu hiệu" khác nhau.

Cách hoạt động

"Vuong" ở vị trí 0: embedding("Vuong") = [0.2, -0.5, 0.8, ...] pos_encoding(0) = [0.0, 1.0, 0.0, ...] ← sin/cos waves input = embedding + pos_encoding = [0.2, 0.5, 0.8, ...] ← token + vị trí "Python" ở vị trí 2: embedding("Python") = [0.7, 0.3, -0.2, ...] pos_encoding(2) = [0.9, -0.4, 0.1, ...] ← khác position 0! → Model phân biệt được "Vuong" đứng trước hay sau "Python"

Python implementation đơn giản

import numpy as np

def positional_encoding(seq_len: int, d_model: int) -> np.ndarray:
    """Tạo positional encoding dùng sin/cos waves.
    seq_len: độ dài sequence (số tokens)
    d_model: kích thước embedding (thường 512 hoặc 768)
    """
    PE = np.zeros((seq_len, d_model))

    for pos in range(seq_len):      # vị trí token
        for i in range(0, d_model, 2):  # từng cặp chiều
            PE[pos, i]   = np.sin(pos / (10000 ** (i / d_model)))
            PE[pos, i+1] = np.cos(pos / (10000 ** (i / d_model)))
    return PE

pe = positional_encoding(seq_len=10, d_model=512)
# shape: (10, 512) — 10 positions, mỗi position 512-dim vector

Understanding Multi-Head Attention

Thay vì dùng 1 attention, Transformer chạy nhiều attention head song song — mỗi head học một loại "mối quan hệ" khác nhau.

Analogy: Đọc 1 câu nhưng đặt nhiều câu hỏi khác nhau cùng lúc:
→ Head 1: "ai là subject?"    → Head 2: "có quan hệ thời gian không?"
→ Head 3: "coreference?"      → Head 4: "quan hệ logic?"
Mỗi head chuyên về 1 loại, kết hợp lại cho hiểu biết phong phú hơn.

Attention là gì? Q, K, V

Attention(Q, K, V) = softmax(Q·Kᵀ / √d_k) · V Q = Query "tôi đang tìm gì?" K = Key "tôi có thể cung cấp gì?" V = Value "nội dung thực sự" Ví dụ: "Con mèo [nó] mệt" Query "nó" hỏi: "ai là tôi?" Key "mèo" trả lời: "tôi liên quan đến đại từ" → attention score cao → lấy Value của "mèo"

Multi-Head = chạy nhiều attention song song

import numpy as np

def attention(Q, K, V):
    """Single attention head."""
    d_k = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d_k)      # similarity scores
    weights = np.exp(scores) / np.exp(scores).sum()  # softmax
    return weights @ V                   # weighted sum of values

def multi_head_attention(X, num_heads=8, d_model=512):
    """8 heads chạy song song, kết quả concat lại."""
    d_k = d_model // num_heads  # 512 // 8 = 64 mỗi head
    outputs = []

    for head in range(num_heads):
        # Mỗi head có weight riêng (học khác nhau)
        Q = X @ np.random.randn(d_model, d_k)  # project xuống 64-dim
        K = X @ np.random.randn(d_model, d_k)
        V = X @ np.random.randn(d_model, d_k)

        head_output = attention(Q, K, V)  # (seq_len, 64)
        outputs.append(head_output)

    # Concat 8 heads: (seq_len, 64*8) = (seq_len, 512)
    return np.concatenate(outputs, axis=-1)

Tại sao Multi-Head tốt hơn Single-Head?

Single Attention
1 "góc nhìn" duy nhất
Dễ bỏ lỡ pattern phức tạp
Multi-Head Attention (8 heads)
8 "góc nhìn" song song
Mỗi head học pattern khác nhau
Kết quả phong phú hơn nhiều

Tổng kết flow đầy đủ

Text input ↓ [Bài 105-106] Tokenization ["Hôm", "nay", "trời"] → [103, 45, 872] ↓ [Bài 108] Token Embedding [103, 45, 872] → 3 vectors (512-dim) ↓ [Bài 109] + Positional Encoding vectors + position info ↓ [Bài 107, 110] Transformer Block × N Multi-Head Attention (8 heads song song) Feed Forward Network Layer Norm ↓ [Bài 103] Linear + Softmax probability distribution over 50,000 tokens ↓ Chọn token tiếp theo → "đẹp"
